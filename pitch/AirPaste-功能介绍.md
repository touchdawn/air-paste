# AirPaste · 功能介绍

> 跨设备共享剪贴板：在一台设备复制，在另一台按一个快捷键（或轻点手机背面）就粘贴。
> 横跨 **macOS / Windows / iPhone**，文本端到端加密，可完全自托管、数据不经任何商业云。

当前版本 **v0.1.1**，真机端到端验证通过。

---

## 一、它解决什么

在手机、Mac、Windows 之间反复搬运同一段内容——验证码、地址、命令、报错、链接、文件——是每天都在发生的小麻烦。

- 用微信「文件传输助手」要切来切去，打断手头的事；U 盘还得插拔。
- 苹果「通用剪贴板」只在苹果设备之间用。
- 商业云剪贴板要把内容上传公有云，公司内容不放心。

**AirPaste 把这件事压成「复制 + 一个快捷键」**，并且三个平台互通、内容端到端加密、可以完全跑在自己内网。

---

## 二、核心功能

### 剪贴板同步
- **文本同步**：复制 → 按发送热键 → 另一台按粘贴热键，双向互通。
- **文件同步**：发送的只是文件「清单」（路径 / 大小 / SHA-256），不预先上传；接收端按下粘贴键时才真正下载，写入系统剪贴板并自动完成一次粘贴。
- **内容去重**：同一份内容重复发送只生效一次，双击热键也不会重复广播。

### 两种工作模式
- **隔离模式（默认推荐）**：收发都显式。只有按下发送热键，当前剪贴板才会被加密外发；远端内容只进 AirPaste 收件箱，**绝不自动覆盖**你的系统剪贴板，按粘贴热键时才借用并随即还原。
- **系统模式**：经典自动同步。每次复制自动外发（带敏感内容过滤），远端内容直接写入系统剪贴板，可直接 `Cmd+V` / `Ctrl+V`。

### 三平台互通
- **macOS**：菜单栏 App，无 Dock 图标。
- **Windows**：系统托盘，可打包成绿色便携版分发给其他机器，免装 Rust 环境。
- **iPhone：免装任何 App** —— 用系统「快捷指令」+「轻点背面」即可收发文本，相当于手机版的发送 / 粘贴热键。

### 安全与加密
- **端到端加密**：文本与文件均为端到端加密（X25519 + XChaCha20-Poly1305），每条内容用随机密钥加密、再为每个可信设备分别封装；服务器自始至终只见密文。
- **设备身份与配对**：每台设备持有 Ed25519 / X25519 密钥，服务器只存公钥；首台设备自动可信，新设备须用 6 位配对码加入。
- **请求签名防重放**：敏感接口要求时间戳 + nonce + 内容哈希 + Ed25519 签名。
- **敏感内容过滤**：自动同步时跳过私钥、JWT、Token、验证码、卡号等。

### 网络与传输
- **控制面 / 数据面分离**：服务器只负责配对、在线状态、信令和元数据，**文件内容默认不经服务器、不落盘**。
- **局域网直连优先**：通过 mDNS 自动发现同网设备，文件点对点直传。
- **加密中继兜底**：直连不可达时走服务器中继，服务器只转发密文帧，不解密、不存储。

### 部署与易用
- **完全自托管**：服务器可跑在自己的机器上（甚至就是收发设备本身），数据不出内网；内嵌 redb 存储，无需外部数据库。
- **托盘一站式**：填服务器地址、生成 / 输入配对码、一键连接；也可在窗口里直接打字发送、拖文件发送、收件箱里点 Copy / Download。
- **贴心打磨**：自定义热键（**按下组合键即录入**，无需手打）、设备命名、开机自启、托盘内直接托管服务器。

## 三、为什么不一样

同类工具不少，AirPaste 的差异集中在三点：

### 1. iPhone 零安装就能接入
不需要在手机上装任何 App。两个系统「快捷指令」绑定「轻点背面」，二击发送、三击取回——这是大多数跨设备剪贴板做不到的，也是现场最直观的一幕。

### 2. 隔离模式：从机制上消除两种不安
剪贴板同步最让人不放心的两件事——**「我每次复制都被发出去了吗」**和**「我正要粘贴的东西会不会被另一台设备悄悄覆盖」**——隔离模式用「收发都必须显式」从设计上根除，而不是靠启发式过滤打补丁。这是产品取舍，也是默认推荐它的原因。

### 3. 端到端加密 + 完全自托管，数据不上任何商业云
针对「公司内容不放心放公有云」这个真实顾虑给出正面回答：中转服务器连明文都看不到，且整套服务可以跑在自己内网。

> 工程上的底子同样扎实：控制面 / 数据面分离、局域网直连优先、加密中继兜底——文件不预先上传、默认不落服务器，既快又安全。整套横跨三平台的工具用纯 Rust 实现，约 11,700 行，一个人用 AI、约 8 天业余时间独立完成，本身就是一次「AI 提效」的实证。
