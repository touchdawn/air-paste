Air Paste 剪贴板同步流程

文本会自动同步到在线可信设备；文件默认只同步清单，真实文件等接收端触发远端粘贴时再下载。

Device A
Server
Device B
Control Plane
复制

A 复制文本

agent 轮询本机剪贴板，默认最多约 750ms 后发现新文本。

准备

过滤并端到端加密

敏感内容过滤通过后，文本被封装成只给可信设备解密的密文。

控制面

Server 保存 clip

server 存储密文和元数据，然后广播 clip.created 事件。

通知

B 收到 WebSocket 事件

在线可信设备收到通知，并用 clip_id 拉取最新内容。

应用

B 写入本机剪贴板

B 解密文本，然后把明文写到自己的系统剪贴板。

用户触发

按 Ctrl+Shift+V

文件默认不会自动下载；接收端先把远端文件清单记为 pending。

数据面

下载并写入文件剪贴板

B 从 A 的 peer 服务下载文件；直连失败时可回退到加密 relay。
