# iPhone 快捷指令接入

iPhone 通过「快捷指令」App 调用服务器的简单设备接口（`/v1/simple/*`），实现**上传剪贴板**和**下载并拷贝**两个动作。无需安装任何 App，也不依赖设备签名。

加密边界：iPhone 上传的文本到达服务器后**立即为所有受信任设备做端到端封装**（X25519 + XChaCha20-Poly1305），落库和广播给桌面端的都是密文——即使某台桌面设备走明文 HTTP 连服务器，线路上也只有密文。明文只存在于两处：iPhone↔服务器这一段（靠 HTTPS 保护）和服务器内存中的 simple 收件箱槽位（供其他简单设备读取，不落盘、重启即清）。

## 1. 服务器端开启简单设备接口

简单设备接口默认关闭，需要配置一个**独立的**令牌（不要复用 `--auth-token`）：

- **托盘应用（内嵌服务器）**：设置面板 → 「简单设备令牌」填一个随机字符串 → 保存并连接。
- **CLI 服务器**：启动时加 `--simple-token <secret>` 或环境变量 `AIRPASTE_SIMPLE_TOKEN=<secret>`。

权限边界：这个令牌**只能**访问 `/v1/simple/*` 两个文本接口，读不了设备列表、加密 clip、文件和中继；反过来主认证令牌也开不了简单接口。

## 2. 桌面端开启镜像（Mac/Windows → iPhone 方向）

文本 clip 是端到端加密的，服务器无法替 iPhone 解密，所以需要桌面 agent 在**显式发送**时额外镜像一份明文到服务器的简单设备收件箱：

- **托盘应用**：设置面板勾选「镜像给简单设备」→ 保存并连接。
- **CLI agent**：启动时加 `--simple-mirror=true`。

只有按 `Option+C` / `Alt+C` 或点「发送」按钮显式发送的文本才会镜像；系统模式下自动发布的剪贴板变化**永远不会**镜像。镜像副本只存在服务器内存里（不落库），服务器重启即清空，且随文本 TTL（默认 10 分钟）过期。

注意：镜像副本是**明文**随发布请求一起发给服务器的。如果某台桌面设备只能用明文 HTTP 连服务器（例如公司网络拦截 TLS），**不要在那台设备上开镜像**——只在与服务器同机或走 HTTPS/可信链路的设备上开。

## 3. 接口说明

两个接口都用 `Authorization: Bearer <简单设备令牌>` 鉴权。

**上传文本**（iPhone → 其他设备）：

```text
POST /v1/simple/clips
Content-Type: application/json

{"text": "要发送的内容", "device_name": "iPhone"}
```

服务器收到后立即为所有受信任设备端到端封装，再作为一条普通加密文本 clip 广播出去：Mac/Windows 端隔离模式下进收件箱（按 `Option+V` / `Alt+V` 粘贴），系统模式下直接写入剪贴板。返回 `{"clip_id": ..., "created_at": ...}`。文本上限 128 KiB，默认 10 分钟过期。（仅当没有任何受信任设备登记过加密公钥时，才退化为明文 clip 并在服务器日志告警。）

**读取最新文本**（其他设备 → iPhone）：

```text
GET /v1/simple/clips/latest
```

返回 `{"text": ..., "source": "TestMac", "created_at": ...}`，没有内容（或已过期）时返回 `null`。内容来源是最近一次桌面端的显式镜像发送，或其他简单设备的上传。

## 4. 创建两个快捷指令

### 「发送剪贴板」（上传）

1. 新建快捷指令，添加操作：
   1. **获取剪贴板**（Get Clipboard）
   2. **获取 URL 内容**（Get Contents of URL）：
      - URL：`http://<服务器地址>:14444/v1/simple/clips`
      - 方法：`POST`
      - 头部：`Authorization` = `Bearer <简单设备令牌>`
      - 请求体：`JSON`，添加字段 `text` = 剪贴板（变量），`device_name` = `iPhone`
2. 命名为「发送剪贴板」。

### 「接收剪贴板」（下载并拷贝）

1. 新建快捷指令，添加操作：
   1. **获取 URL 内容**：
      - URL：`http://<服务器地址>:14444/v1/simple/clips/latest`
      - 方法:`GET`
      - 头部：`Authorization` = `Bearer <简单设备令牌>`
   2. **获取词典值**（Get Dictionary Value）：键 `text`
   3. **拷贝到剪贴板**（Copy to Clipboard）：内容为上一步的词典值
2. 命名为「接收剪贴板」。之后回到目标 App 长按粘贴即可。

### 触发方式建议

- **轻点背面**：设置 → 辅助功能 → 触控 → 轻点背面，轻点两下绑「发送剪贴板」、轻点三下绑「接收剪贴板」——体验上最接近桌面端的 `Option+C` / `Option+V`。
- **共享表单**：给「发送剪贴板」加一步「获取共享表单输入」，就能在任意 App 选中文字后直接通过共享菜单发送（连复制都省了）。
- 也可以放主屏幕图标或锁屏小组件。

## 5. 网络与安全注意

- iPhone 与服务器要能互通：同一局域网用 `http://<内网IP>:14444`；外网访问强烈建议套 HTTPS 反向代理（快捷指令完全支持 `https://`），明文 HTTP 只建议在可信局域网内使用。
- 令牌保存在快捷指令里，设备丢失请及时更换 `--simple-token`。
- iOS 限制：快捷指令无法后台监听剪贴板，收发都是手动触发；「获取剪贴板」首次运行会请求权限，允许即可。
- 当前只支持文本。文件传输依赖设备签名和点对点下载，快捷指令做不了，后续需要 Scriptable 或原生 App。
