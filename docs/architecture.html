AirPaste 架构学习图

这版把图拆成 6 个步骤。默认只显示当前步骤相关的箭头，先看控制面，再看 mDNS，再看文件直连，避免所有线一起出现。

Step 1/6

怎么读这张图

  1. Server 是控制面。它负责设备注册、信任、clip 元数据和 WebSocket 通知。
  2. 文本内容走 server。但文本在 agent 端已经端到端加密，server 保存的是密文。
  3. 文件内容不走 server。server 只保存文件 manifest，真正字节由接收端从源 agent 下载。
  4. mDNS 不是传文件。它只帮接收端发现源设备在局域网里的 IP:port。

代码入口

agent runtime / clipboard flow：crates/airpaste-agent/src/main.rs mDNS discovery：crates/airpaste-agent/src/discovery.rs peer file server：crates/airpaste-agent/src/peer.rs server REST routes：crates/airpaste-server/src/routes.rs server WebSocket：crates/airpaste-server/src/ws.rs shared API types：crates/airpaste-protocol/src/api.rs

当前版本 relay 数据通道还没实现，所以跨网段文件传输依赖 --peer-public-url 或可达的源设备地址。
